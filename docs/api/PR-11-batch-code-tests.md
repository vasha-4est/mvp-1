# PR-11 — Manual test vectors (curl) for batch.code generation `B-YYMMDD-NNN`

Assumptions:
- Next.js endpoint: `POST https://<YOUR_VERCEL_DOMAIN>/api/batch/create`
- Header auth: `x-gas-api-key: <GAS_API_KEY>` (or whatever header your implementation uses)
- Request may include: `request_id` (uuid), `note`, `meta`.
- Client MUST NOT send `code`. Server returns it.

Notes:
- Route Handlers parse JSON with `await request.json()`; use `Content-Type: application/json`.
- `batch.code` format: `B-YYMMDD-NNN` where `NNN` is daily sequence.
- Idempotency: same `request_id` must NOT create new row and must NOT bump sequence.

---

## 1) Create batch (no client code)

```bash
curl -sS -X POST "https://<YOUR_VERCEL_DOMAIN>/api/batch/create" \
  -H "Content-Type: application/json" \
  -H "x-gas-api-key: <GAS_API_KEY>" \
  -d '{
    "request_id": "11111111-1111-1111-1111-111111111111",
    "note": "smoke",
    "meta": { "source": "preview" }
  }' | jq
```

Expected (example):
- HTTP 201
- JSON:

```json
{
  "ok": true,
  "data": { "id": "<batch_id>", "code": "B-250215-001", "status": "created", "created_at": "..." },
  "replayed": false
}
```

Acceptance:
- `batch.code` exists
- matches regex: `^B-\d{6}-\d{3}$`

---

## 2) Replay same request_id (idempotency)

```bash
curl -sS -X POST "https://<YOUR_VERCEL_DOMAIN>/api/batch/create" \
  -H "Content-Type: application/json" \
  -H "x-gas-api-key: <GAS_API_KEY>" \
  -d '{
    "request_id": "11111111-1111-1111-1111-111111111111",
    "note": "smoke",
    "meta": { "source": "preview" }
  }' | jq
```

Expected:
- HTTP 200
- JSON contains `replayed: true`
- same `data.id` and same `data.code`
- sequences for the day are not incremented

---

## 3) Create second batch (new request_id)

```bash
curl -sS -X POST "https://<YOUR_VERCEL_DOMAIN>/api/batch/create" \
  -H "Content-Type: application/json" \
  -H "x-gas-api-key: <GAS_API_KEY>" \
  -d '{
    "request_id": "22222222-2222-2222-2222-222222222222",
    "note": "smoke"
  }' | jq
```

Expected:
- HTTP 201
- `batch.code` increments: `...-002` (same `YYMMDD`)

## 4) Reject client-supplied code

```bash
curl -sS -X POST "https://<YOUR_VERCEL_DOMAIN>/api/batch/create" \
  -H "Content-Type: application/json" \
  -H "x-gas-api-key: <GAS_API_KEY>" \
  -d '{
    "request_id": "33333333-3333-3333-3333-333333333333",
    "code": "B-000000-999",
    "note": "smoke"
  }' | jq
```

Expected:
- HTTP 400 and message similar to: `Field 'code' is not allowed; code is generated by the server`

Acceptance:
- returned batch code is never client-controlled

---

## 5) Concurrency smoke test (10 parallel creates)

```bash
for i in $(seq 1 10); do
  rid=$(uuidgen | tr "[:upper:]" "[:lower:]")
  curl -sS -X POST "https://<YOUR_VERCEL_DOMAIN>/api/batch/create" \
    -H "Content-Type: application/json" \
    -H "x-gas-api-key: <GAS_API_KEY>" \
    -d "{
      \"request_id\": \"${rid}\",
      \"note\": \"smoke\"
    }" &
done | cat

wait
```

Expected:
- 10 distinct codes (no duplicates)
- Codes follow the same day `YYMMDD`
