# Contributing

## PR-only workflow
- All changes must be submitted as a Pull Request (PR).
- Do not push direct changes to `main`.
- Keep each PR scoped to one task.

## Review and approval
- Owner review is done through PR comments and approvals.
- Address review feedback in follow-up commits on the same PR.

## Required checks
- Vercel Preview must be created and pass before merge.
- Confirm the Preview URL loads successfully.
